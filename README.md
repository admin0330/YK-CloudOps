# ym1r

`ym1r` 是一个面向展示、运维和个人内容聚合的站点。它把首页做成运维中心，把个人主页、项目和 GitHub 收到次级入口里，整体视觉偏 Apple / Liquid Glass。

线上地址：<http://8.162.24.188:3001>

## 这个仓库展示什么

- 一个清晰的运维首页，而不是把聊天放在主入口
- 一个独立的次级入口页，用来承接个人主页、项目和 GitHub
- 一个 AI 运维助手入口，用于排障、命令建议和日志解释
- 一套统一的浅色 / 深色主题与中英文切换
- 一个同时适配桌面和手机浏览器的品牌型界面

## 页面地图

### `/` 运维首页

- 首页直接进入运维中心，而不是 AI 对话页
- 上方导航只保留最核心的入口
- 首屏提供状态感、引导感和清晰的视觉层级
- 中间强调服务器管理、权限审计和 AI 运维辅助
- 底部提供次级入口提示，把个人内容放在更合适的位置

### `/portal` 次级入口页

- 个人主页、项目和 GitHub 收在同一页
- 这个页面不抢首页的主题，更适合用来浏览细节
- 用于表达“想了解我”和“想看项目”的访问路径

### `/cloudops` AI 运维助手

- 用于日志分析、错误解释和排障建议
- AI 不是主入口，而是运维场景中的辅助工具
- 更适合处理小故障、命令建议和运维简报

### `/admin` 管理页

- 专门面向管理任务的独立页面
- 用于用户、文件、服务器和数据类操作
- 移动端做了更紧凑的间距处理，尽量避免覆盖和挤压

## 视觉语言

- Apple / Liquid Glass 风格
- 统一的 pill 按钮、glass card 和柔和阴影
- 更克制的空间和层级
- 手机端默认收紧，尽量不让 UI 重叠
- 切换动画保持线性，前半段更快，后半段更慢

## 交互特点

- 中英文切换会立即影响全站文案
- 浅色 / 深色切换用线性遍布式过渡
- 主站和次级入口分层清楚
- 移动端菜单是轻量展开，不押住内容
- 整体细节更像一个可直接浏览的产品页，而不是内部文档

## 适合的观众

- 想看一个运维中心风格首页的人
- 想把个人主页、项目和管理入口分开的人
- 想让 AI 只做辅助而不抢主线的人
- 想要一套可以让别人直接看懂的站点结构的人
